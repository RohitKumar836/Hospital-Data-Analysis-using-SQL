# Hospital-Data-Analysis-using-SQL
Analyzed hospital emergency room dataset using SQL to understand patient trends, wait times, and satisfaction levels


 **Description**

This project focuses on analyzing hospital emergency room (ER) data using SQL to uncover insights related to patient flow, wait times, and satisfaction levels. The dataset consists of over 9,000 patient records, including demographic details, admission status, department referrals, wait time, and satisfaction scores.

The primary goal of this project is to demonstrate strong SQL skills in data cleaning, transformation, and analysis while working with real-world messy data.

 **Key Objectives**

 
Analyze patient distribution and demographics
Evaluate hospital performance using wait time and satisfaction metrics
Identify high-load departments and operational bottlenecks
Understand patterns in patient admissions and service efficiency


 **Data Cleaning & Challenges**

During the project, several real-world data issues were encountered and resolved:

Handled file import restrictions (secure_file_priv) in MySQL
Resolved data type mismatches (e.g., TRUE/FALSE values in numeric columns)
Managed missing and empty values in satisfaction scores
Fixed column size limitations (data too long errors)
Worked with MySQL safe update mode restrictions

** Analysis Performed**


Total number of patients and admission analysis
Average wait time and satisfaction score
Gender distribution of patients
Department-wise patient load
Department with highest wait time
Age group analysis for specific departments
Wait time comparison across different patient groups


** Key Insights**


Average patient wait time is approximately 35 minutes
Certain departments (e.g., Neurology) have higher wait times
Patient satisfaction tends to decrease with increased wait time
Patient distribution is nearly balanced across gender
Specific age groups contribute more to certain departments
